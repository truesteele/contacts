### Iteration 2 — US-002: Pitches API — PATCH + POST (2026-04-11)
**Story:** Extend pitches API with bookmark/notes in PATCH and new POST for pitch creation
**Status:** COMPLETE
**What was done:**
- PATCH handler: added `is_bookmarked` (boolean) and `user_notes` (string) to accepted fields
- POST handler: accepts `podcast_target_id` + `speaker_slug`, resolves speaker_profile_id from slug
- POST upserts: checks for existing pitch by podcast+speaker combo, returns existing ID if found
- Creates minimal pitch record (no fit data) with timestamps
- Typecheck passes clean
**Learnings:**
- Speaker profiles use `slug` field for lookup, id is internal
- Supabase `.single()` returns error when no rows found — use that for existence check
- POST returns `{ id, pitch_id }` for consistency with PATCH responses

### Iteration 3 — US-003: Discover API — Bookmark/notes + filter (2026-04-11)
**Story:** Return bookmark/notes in pitch data and support bookmarked filter
**Status:** COMPLETE

- Added `is_bookmarked, user_notes` to pitch select fields in speaker-driven path

- Typecheck passes clean

- The `bookmarked` filter only applies in the speaker-driven path since pitches are per-speaker

### Iteration 4 — US-004: Generate API — Opus 4.6 with full voice guides (2026-04-11)
**Story:** Upgrade pitch generation to Opus 4.6, embed comprehensive voice guides, add force/auto-create
**Status:** COMPLETE
**What was done:**
- Changed MODEL from `claude-sonnet-4-6` to `claude-opus-4-6`
- Replaced abbreviated voice guides (~8 lines each) with comprehensive versions (~50+ lines each):
  - Sally: scene-first openings, named people rule, contrast signature, mantras, minister's cadence, em dash policy (max 1), vocabulary, email patterns, OC descriptions
  - Justin: punchy 10-20 word sentences, "would love" construction, builder identity, no em dashes ever, relationship-first approach, no-pressure framing, vocabulary
- Expanded AI_WRITING_RULES with additional banned patterns from Signs of AI Writing doc
- Added `force?: boolean` to GenerateRequest — when true, skips `!p.pitch_body` filter to allow regeneration
- Auto-create pitch record: when `podcast_id` + `speaker_slug` provided and no pitch exists, inserts minimal row before generating
- Added `podcast_profile` to podcast_targets select — user prompt now includes about, hosts, audience, format when available
- Subject line target tightened from 60 to 50 chars per outreach best practices
- Typecheck passes clean
**Learnings:**
- `podcast_profile` is a JSONB column on `podcast_targets` — structure varies but commonly has about, hosts, audience fields
- The detail API already selects `*`, so podcast_profile was already available there — just needed adding to generate route's select
- Force flag applies at the toGenerate filter step, keeping the rest of the generation pipeline unchanged
- Auto-create mirrors the pattern from the pitches POST handler (US-002)
