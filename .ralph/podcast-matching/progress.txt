## US-005: Composite Multi-Signal Scoring ✓
Date: 2026-04-11
Commit: 08ddf84

**What was done:**
- Modified `scripts/intelligence/score_podcast_fit.py` to add 5-signal composite scoring
- Signals and weights:
  1. GPT fit score (35%) — existing structured output from GPT-5.4 mini
  2. Embedding similarity (30%) — cosine similarity between speaker profile_embedding and podcast description_embedding
  3. Similar-speaker boost (15%) — 0.15 if 'similar_speaker' in discovery_methods, else 0.0
  4. Activity recency (10%) — based on last_episode_date (1.0 for <7d, down to 0.1 for >90d)
  5. Episode count signal (10%) — based on episode_count (1.0 for 200+, down to 0.1 for <10)
- Composite tiers: strong >= 0.70, moderate >= 0.45, weak < 0.45
- Signal breakdown saved in `topic_match` JSONB with keys: composite_score, signals, matching_pillars, discovery_methods
- Added `--no-composite` flag for backward-compatible GPT-only scoring
- Pure Python cosine similarity (no numpy dependency — numpy was broken in venv, x86_64 on arm64)
- Loads speaker embedding from speaker_profiles and podcast embeddings from podcast_targets via psycopg2
- Bulk loads embeddings and discovery_methods for all podcasts in one query (efficient)
- Missing embeddings handled gracefully with neutral default (0.3)

**Test results (--speaker sally --limit 10 --test):**
- All 10 podcasts scored with 5 signals visible
- Composite tiers differ from GPT-only: e.g., "Alpine Camp Podcast" GPT=0.31 but composite=0.26 (weak) due to low recency/episodes
- "Therapy in the Great Outdoors": GPT=0.84 (strong), composite=0.57 (moderate) — embedding=0.55 + low recency pulled down
- `--no-composite` produces GPT-only scores (backward compat confirmed)
- Missing embeddings return 0.3 default (no crashes)

**Verified:**
- `--speaker sally --limit 10 --test` shows composite scores with all 5 signals ✓
- Output includes signal breakdown per podcast ✓
- Composite tiers differ from GPT-only for some podcasts ✓
- `--no-composite` flag produces GPT-only scores ✓
- Scores saved with signal breakdown in `topic_match` JSONB ✓
- Missing embeddings handled gracefully ✓

**Notes:**
- numpy was broken in venv (x86_64 binary on arm64 mac) — used pure Python math.sqrt + dot product instead
- psycopg2 `::text` cast on vector columns returns string format `[0.1,0.2,...]` — easy to parse
- Bulk-loading embeddings for all podcasts in one query is much faster than per-podcast queries
- The `_signals` key on the score dict is a display-only convenience (not saved to DB)
- `topic_match` column changed from array (matching_pillars) to object (with signals) — backward compat for existing rows is fine since the UI reads both formats
