### Iteration 3 — US-003: Create API Route for Project Tasks CRUD (2026-03-11)
**What:** Created CRUD API route at `/api/projects/[projectId]/tasks` for project task management.
**How:**
- Created `app/api/projects/[projectId]/tasks/route.ts` with GET, POST, PATCH, DELETE handlers
- GET: fetches all tasks for a project, ordered by section + sort_order, returns both flat array and grouped-by-section object
- POST: creates a new task with project_id, section, title, etc.
- PATCH: updates allowed fields (status, notes, owner, title, description, due_date) with auto-updated_at timestamp
- DELETE: removes a task by id, scoped to project_id
- All handlers use `createProjectsServerClient()` (service role key)
- Typecheck passes clean
**Learnings:**
- Next.js 15 dynamic route params are `Promise<{ projectId: string }>` — must `await params`
- Middleware redirects all non-public paths to `/login` — API route test returns redirect until US-004 adds `/projects` to public paths
- Pattern: filter updates to allowlisted fields to prevent arbitrary column writes
